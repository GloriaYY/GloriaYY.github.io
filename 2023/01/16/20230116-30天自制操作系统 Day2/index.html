对makeFile的认识

还是follow书籍，并且也参考别人的笔记，还是这位大佬的笔记做的很详细，把链接放在这里：https://blog.csdn.net/qq511714326/article/details/82915012

既然别人已经做了很详细的笔记了，我就不用再重复了。学习中遇到了一些问题，和大家分享一下。

最开始我按照书里的描述，把文件copy到helloos4下面，执行make -r helloos.img的时候，报错说 no rule. make stop.

我于是打开make.bat，发现确实没有rule， 内容就是…\z_tools\make.exe %1 %2 %3 %4 %5 %6 %7 %8 %9

心里升起一个大谜团，这么多个%和数字列出来，是什么意思呢？百度搜索了一下，最终找到一篇还不错的博文，讲得比较系统，链接留在这里：https://www.cnblogs.com/lsgxeva/p/10694546.html

原来%1-%9的意思是这个批处理命令读入9个参数的意思。好吧，这里暂时就懂了。

MakeFile的内容结合作者的描述大致也猜出来了，MakeFile就是把原来单个的批处理命令合成到一起，也可以用来生成文件

MakeFile的源码：

// 
#默认

default :
    ../z_tools/make.exe img

#文件生成规则

ipl.bin : ipl.nas Makefile
    ../z_tools/nask.exe ipl.nas ipl.bin ipl.lst

helloos.img : ipl.bin Makefile
    ../z_tools/edimg.exe   imgin:../z_tools/fdimg0at.tek \
        wbinimg src:ipl.bin len:512 from:0 to:0   imgout:helloos.img

# 命令

asm :
    ../z_tools/make.exe -r ipl.bin

img :
    ../z_tools/make.exe -r helloos.img

run :
    ../z_tools/make.exe img
    copy helloos.img ..\z_tools\qemu\fdimage0.bin
    ../z_tools/make.exe -C ../z_tools/qemu

install :
    ../z_tools/make.exe img
    ../z_tools/imgtol.com w a: helloos.img

clean :
    -del ipl.bin
    -del ipl.lst

src_only :
    ../z_tools/make.exe clean
    -del helloos.img

从源码看的话，我猜如果是生成文件，冒号前面就是一个文件名，如果是整合命令，冒号前面就是命令名。生成文件的话，冒号后面同一行的内容就是依赖文件，下一行开始是生成文件的操作，后面有输出文件。如果是命令的话，冒号后面同一行里就不可以有内容了，下一行开始写命令的执行语句。

寻找make之源

MakeFIle算是想明白了吧，但是make是什么内容还是看不到。书里说make是GNU团队的作品，于是继续百度，啊哈，找到了，在这里：http://www.gnu.org/software/make/

扫了一眼doc，发现我的理解还是正确的吗。其实查找之前IDA里面试图看看源码来着，结果好像不行啊，UPX1这个指令IDA也不认识，后面继续读原团队的文档比较好吧。后时候，动作太快也不是件好事情。
