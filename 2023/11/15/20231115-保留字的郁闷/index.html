保留字的郁闷

发表于2023-11-15|更新于2023-11-15
|浏览量:

1064 Error

又是把数据导入库里的时候，发生了错误，1064。

ProgrammingError: (1064, "You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near '(30),titleVARCHAR(30),contentVARCHAR(30))' at line 1"

排查一遍，发现是表格字段使用了保留字。我其实不太喜欢这样的报错信息，就不能直接一点吗，就说you have used reserved words which caused SQL syntax error不行吗？

然后，保留字我看了一下，目测至少有二三百个吧，真是郁闷啊，敢不敢再多一点保留字，真的是把常用词都霸占了的感觉。

Anyway， 还是把保留字的查询连接存一下吧，看起来也是高频使用信息呢

官方文档：https://docs.oracle.com/cd/E17952_01/mysql-8.0-en/keywords.html#keywords-8-0-detailed-B

保留字的必要性

Mysql有这么多的保留字，我也是真的没有想到啊，不得不说，作者还真是挺偷懒的，为什么不把常用字让渡给用户，非要保留给程序，我们现在设计个字段，还得研究有没有用到保留字，而且是上百个保留字啊，想着都心累~~
