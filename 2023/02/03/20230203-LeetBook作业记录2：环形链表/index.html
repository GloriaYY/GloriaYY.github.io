作业感受

检查空指针很重要，空指针无法参与运算，每次做比较条件时都需要校验参与比较的两数是否非空。

之前设计的题目ListNode还可以修改，加一些自定义的属性，但是这次应该用的是Java的内置库，我试过修改ListNode的属性，但是都报错了。

作业代码

/**
 * Definition for singly-linked list.
 * class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode(int x) {
 *         val = x;
 *         next = null;
 *     }
 * }
 */

public class Solution {
    
    public boolean hasCycle(ListNode head) {

        int pos = -1; 
        if(head == null) {
            return false;
        }
            
        ListNode slow = head;
        ListNode fast = head.next;
        while( slow !=null && fast != null){
            if(slow == fast) return true;
            else {
                slow = slow.next;
                pos++;            
                
                if( fast.next != null){
                    fast = fast.next.next;
                }else return false;
            }
                     
        }
        pos = -1;
        return false;
        
    }    
}

作业成绩

作业改错

上面的作业是环形链表，检查循环似乎是可以过关的，但是其实逻辑上还是有漏洞，没有找到正确的环入口，pos的值不正确。后面做环形链表II时就暴露出了这个问题。

我一开始的思路时找到环之后，通过一个循环语句让快慢指针同步走，然后删掉慢指针的节点，知道两个节点不等时，返回快指针对应的节点，但是很遗憾，这个办法超时了。

后来，我在评论区看到了一个思路清晰的解法，发现了自己的错误: 第一， 快慢指针的起点不一致，导致量化比较困难；第二，没有进行严谨的数学分析，思维上偷懒了。

评论的代码思路很清晰，摘录如下：

public class Solution {
    public ListNode detectCycle(ListNode head) {
        ListNode slow = head, fast = head;

        while (slow != null && fast != null && fast.next != null){
            fast = fast.next.next;
            slow = slow.next;
            if(slow == fast) break;
        }
        //如果满足下面的条件，说明无环
        if(fast == null || fast.next == null) return null;

        //下面开始找环的入口节点  
        //刚才的循环 慢指针走了a + b (a是从开始到入口节点) 假设等于cnt
        //          快指针走了 a + b + c + b (b + c = 环)，应该等于2 * cnt
        //推导出 a = c，所以把慢指针再放到头节点，下一次快慢指针相遇的节点即环的入口节点
        slow = head;
        while (slow != null && fast != null){
            if(slow == fast) break;
            slow = slow.next;
            fast = fast.next;
        }
        return fast;
    }
}
