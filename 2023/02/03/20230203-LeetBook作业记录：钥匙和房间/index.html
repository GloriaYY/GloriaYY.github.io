题目概述

队列和栈章节，小结的最后一道题。题目给了一个房间数组，每一个房间里都有一串钥匙，钥匙数量在0-n之间不等，一开始只有0号房间的钥匙，题目要求返回是否能遍历所有房间

题目地址：https://leetcode-cn.com/leetbook/read/queue-stack/gle1r/

一、题目思路

这道题一开始我还在幻想不用dfs， 用一个循环和hashmap搞定，然而并不能够，因为顺序遍历房间虽然可以把有钥匙的都标识出来，但是用钥匙开房间却不是一个顺序的过程，需要根据钥匙的情况逐步递进的开房门。

所以还是得用dfs， 因为代码写的不熟练，出错了很多次，最后发现有一个语句下标引用错了，提交了大概7、8次吧。什么时候才能像大佬一样在纸上写完代码一次就通过呢？我好像很难在纸上思考。

二、作业记录

1.作业代码

代码如下：

class Solution {
    private HashMap hasKey = new HashMap<Integer,Boolean>();

    public boolean canVisitAllRooms(List<List<Integer>> rooms) {

        // 0号房间默认开着；
        hasKey.put(0,true);

        //进入0号房间拿钥匙
        open( rooms, 0);
        return hasKey.size() == rooms.size();
    }

    public void open( List<List<Integer>> rooms, Integer room){
  //房间里没有钥匙，或者没有这个房间的钥匙，什么都不做。
        if( room == null || !hasKey.containsKey(room))
           return;
       //房间里有钥匙，启动开锁流程
        for( int j : rooms.get(room)){
            if( !hasKey.containsKey(j)){
                hasKey.put( j, true);
                open( rooms, j);
            }            
        }    
    }
}

有几次错误是因为hasKey没有作为全局变量，好像更新起来就有一些问题。这个错误需要记住！我猜应该是递归的过程中如果不是全局变量的话，变量更新的时间会延迟，所以会出现结果错误的情况。

复杂度分析

时间复杂度：基本上就是O(n),因为实际上就是把各个房间的钥匙找到的过程，找钥匙本身是O(1)的操作，所以总体的时间复杂度和房间总数成正比。

空间复杂度：因为用了HashMap额外存储了钥匙，复杂度可以说近似与O(n), n = riooms.size();
