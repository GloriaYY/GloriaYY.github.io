Python的函数默认值情况下会出一些问题

话说一年多没写一条代码了，最近一个月基本上工作日都能写写代码，还是要感谢单位的培训课程。

基本上都是Python的课程，所以也就没得选，没得挑了，谁让流行趋势时Python呢。但是其实Python的函数用起来方便的同时，小毛病也是很隐藏的。

前几天下载了一些数据，先保存成csv文件了，今天读取出来才发现，多了一列’Unnamed:0‘， 很难理解这样设计函数是几个意思。最要命的是这多出来的一列并不是索引，很难引用。(后来发现居然可以引用，就是注意Unnamed：后面有一个空格才是0）

这里记录一下我的处理方法，遇到同类问题的大家可以借鉴一下哈。我导入的数据命名为f

// An highlighted block
# 去掉Unnamed:0这列
f = f[f.columns[1:]]# 这行是发现索引Unnamed： 0可以引用后修改的代码
# 重写数据
f.to_csv('f.csv',index=False)
# 重读数据
f = pd.read_csv('f.csv', encoding='gbk')
# 再检查是否还有’Unnamed:0‘这个多余的列
f.head()

实测有效。
