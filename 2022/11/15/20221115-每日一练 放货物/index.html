每日一练 放货物

发表于2022-11-15|更新于2022-11-15
|浏览量:

今天在每日一练里练习算法，也是好久没有练了。

打开一道题目，标明难度为困难，类别为数学。

其实题目初看完全不困难啊，很快解决了的感觉，但就是无法通过所有测试用例。而我就是完全看不出来哪里有问题。

只能通过80%的测试用例，想不出来另外20%的用例长什么样子啊~~


从上图的示例和题目的代码来看，每一行输入后，都会执行计算，得出一个最优解。所以其实这个问题就简化成了一行两个数，计算商的向上取整值，13是被禁用的除数。 我完全没有看到困难在哪里——直到我的代码只能通过80%的测试用例！

冥思苦想了很久，发现bug出现在没有很好的处理剩余非整车装的货物与13这个数字之间的关系，于是加了两行代码，终于，成功通过所有测试用例。

代码如下。 难点就在于13这个数字的处理上。但是标注为困难，其实还是有些夸张了，总体上应该属于简单或者中等吧，毕竟算法练习少得可怜的我居然也做出来了呢！

// solution函数的代码是我的，其他的代码是系统给定的
import java.util.ArrayList;
import java.util.Scanner;

class Main {
    public static void main(String[] args) {
        Scanner scan = new Scanner(System.in);

        while (scan.hasNext()) {

            String str_0 = scan.nextLine();
            String[] line_list_0 = str_0.trim().split(" ");
            ArrayList<Integer> arr = new ArrayList<>();
            for (int i = 0; i < line_list_0.length; i++) {
                arr.add(Integer.parseInt(line_list_0[i]));
            }
            int result = solution(arr);
            System.out.println(result);
        }
        scan.close();

    }

    public static int solution(ArrayList<Integer> data) {
        int result = 0;

        // TODO: 请在此编写代码
        int n = data.get(0);
        int k = 1;
        
        // 处理货车载荷不能为13的约束条件
        if( data.get(1) == 13){
            k = data.get(1) - 1;
        }else{
            k = data.get(1);
        }
        
        // 排布货物，注意剩余的非整车装的货物也不能是13！
     
        if( n % k == 0){
            result = n / k;
        }else if( n % k == 13){
            result = n / k + 2;
        }else{
            result = n / k + 1;
        }        
        

        return result;
    }
}
