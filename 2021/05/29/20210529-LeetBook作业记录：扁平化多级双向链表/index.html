LeetBook作业记录：扁平化多级双向链表

发表于2021-05-29|更新于2021-05-29
|浏览量:

作业感受

题目地址： https://leetcode-cn.com/leetbook/read/linked-list/fw8v5/

这个作业的思路其实说出来很清晰，就是遍历链表的时候，遇到有child指针的节点就下钻一层，将子链表插入到上层链表中。核心是要拿到子链表的尾节点。

但这道题其实实现起来是有些难的，一眼看过去就是一定要递归，但是自己用双指针的方法写了几个版本，但是都会出问题。两级链表还好，但是三级链表的时候就会丢掉尾部。

于是查看了题解，但是说实话，官方的题解写的好难理解，其实二叉树的概念对理解问题完全没有帮助的感觉。

在评论区找到一个优雅的递归解法，在我看来这个方法用private一个全局变量Node的方法变相实现了双指针，完美的处理掉我遇到的问题。看来变量的设置也要灵活，不能被大括号局限了自己的思维啊！

自己的代码

/*
// Definition for a Node.
class Node {
    public int val;
    public Node prev;
    public Node next;
    public Node child;
};
*/

class Solution {
    public Node flatten(Node head) {

        if(head == null) return null;

        Node p1 = head, p2 = p1.next , sub ;
        
        // 寻找有子节点的节点
        while( p1.child == null){
            p1 = p1.next;   
            if(p1 == null) return head;
        }

       //标识子链表首节点，处理子链表头部与母链表的关系
        sub = p1.child;
        sub.prev = p1;
        p2 = p1.next;
        p1.next = sub;
        p1.child = null;
        
        // 遍历子链表
        while(sub.next != null){
           sub = sub.next;
           if(sub.child != null){
               sub.next = flatten(sub.child);
               sub.child = null;
           }

        }

        // 处理子链表尾部与母链表的关系
        p2.prev = sub;
        sub.next = p2;
       
        return head;        
        
    }
}

评论区解法

class Solution {
   // 虚拟节点， 我理解是用来存放子链表的尾部的
    private Node node;
    
    public Node flatten(Node head) {
        if(head == null) {
            return null;
        }
        
        //这里其实是用来处理子链表尾部的节点next为空时，链接到上级链表。因为node是nodeNext的前序节点，flatten(head.child)最后执行到最后一个子链表的最后一个节点的时候，child指针已经都变成next了，这个时候node是子链表的走一个节点，head这个时候是上一级链表有child指针的节点的nextNode
        if(node != null) {
            node.next = head;
            head.prev = node;
            node.child = null;
        }
      
     // 这里我理解是模拟了之前常用的双指针
        node = head;
             //开始遍历child时（比如3, 7)，next会被改变，所以用nextNode保存一下（原注释）
        Node nextNode = head.next;
     
     // 递归用来遍历子链表，child方向优先
        flatten(head.child);
        
        //这个递归用来遍历母链表，next方向次之
        flatten(nextNode);
        
        return head;
    }
}

给评论区的解法添加了几条注释。原来只有一条，看得不太明白。确实要佩服这位大神！大神的昵称是shuangc。很优雅的解法。学习啦！

我终于明白了为什么要使用两个递归语句了，其实这道题递归的终止条件就是返回null，这也就是flatten函数可以模拟dfs的原因。

两个语句就是向两个方向遍历链表，这个思路其实和我的完全一致，但我的代码就是那么水，哎，多写吧，批评自己也于事无补啦。

官方题解

/*
// Definition for a Node.
class Node {
    public int val;
    public Node prev;
    public Node next;
    public Node child;

    public Node() {}

    public Node(int _val,Node _prev,Node _next,Node _child) {
        val = _val;
        prev = _prev;
        next = _next;
        child = _child;
    }
};
*/
class Solution {
  public Node flatten(Node head) {
    if (head == null) return head;
    // pseudo head to ensure the `prev` pointer is never none
    Node pseudoHead = new Node(0, null, head, null);

    flattenDFS(pseudoHead, head);

    // detach the pseudo head from the real head
    pseudoHead.next.prev = null;
    return pseudoHead.next;
  }
  /* return the tail of the flatten list */
  public Node flattenDFS(Node prev, Node curr) {
    if (curr == null) return prev;
    curr.prev = prev;
    prev.next = curr;

    // the curr.next would be tempered in the recursive function
    Node tempNext = curr.next;

    Node tail = flattenDFS(curr, curr.child);
    curr.child = null;

    return flattenDFS(tail, tempNext);
  }
}
