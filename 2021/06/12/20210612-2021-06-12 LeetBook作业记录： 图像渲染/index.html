题目概述

题目要求把与给定pixel四个方向连接的颜色相同的pixel涂成新的颜色。颜色用正整数表示。

题目地址：https://leetcode-cn.com/leetbook/read/queue-stack/g02cj/

一、解题思路

这道题和岛屿数量这道题基本上是类似的，思路就是用dfs。如果相邻四个方向的pixel都和起点pixel的颜色一致，就涂上新颜色，否则什么都不做。涂色之后，重复检查新涂色pixel相邻四个方向的Pixel，重复刚才的操作。

踩了几个坑，所以这里补一下思路的注意事项：
1）newColor不一定是一种不一样的颜色，有可能和image[sr][sc]一个颜色。把这个测试用例po在这里，有没有觉得很坑：

2）行列的下标一定要对应，我有一次把行列的下标标反了，浪费了一次提交。

二、作业记录

1.作业代码

代码如下：

class Solution {

    public int[][] floodFill(int[][] image, int sr, int sc, int newColor) {

        int oldColor;
        oldColor = image[sr][sc];
        
        if( oldColor == newColor)
            return image;
           
        refill(image, sr, sc, oldColor, newColor);
        return image;

    }

    public void refill( int[][]image, int sr, int sc, int oldColor, int newColor){

        if( sr < 0 || sc < 0 || sc >= image[0].length || sr >= image.length || image[sr][sc] != oldColor)
           return;
        
        image[sr][sc] = newColor;  
        
        refill(image, sr-1, sc, oldColor, newColor);
        refill(image, sr+1, sc, oldColor, newColor);
        refill(image, sr, sc-1, oldColor, newColor);
        refill(image, sr, sc+1, oldColor, newColor);      
    }
}

复杂度分析

时间复杂度：refill函数的操作基本上就是赋值，应该是O(1)复杂度，我们用到了递归从开始节点向四个方向搜索，基本上可以近似于四叉树，我想是不是应该是log以4为底N的对数，N是image的所有元素的个数。这么说起来这个solution的时间复杂度可以说是O(logN)。

空间复杂度：基本上没有利用额外空间存储元素，就是O(1)了吧。
